## 第2弾：URL 解析・座標特定（2026-02-23）

### 要件
- [ ] Google Maps 短縮 URL から住所と座標を自動抽出する
- [ ] URL の例：`https://maps.app.goo.gl/K52RyZTAyt6wGpVf8?g_st=ic`
- [ ] 解析後、住所（例：鹿児島県鹿児島市下荒田３丁目３８−１８）と座標を特定する
- [ ] 座標特定が正確に行われること

### 実装内容
- **geo.resolveMapUrl ルーター** — Google Maps 短縮 URL をリダイレクト先に解決
- **住所抽出** — URL クエリパラメータから住所を抽出
- **Nominatim ジオコーディング** — 住所から座標を取得
- **全角→半角変換** — 日本語住所の数字を半角に統一
- **curlフォールバック** — Node.js fetch がブロックされる場合、curl コマンドで対応

### 実装済み機能
- ✅ Google Maps 短縮 URL の解析
- ✅ 住所の自動抽出
- ✅ 座標の正確な特定（緯度経度）
- ✅ 施設名の抽出（例：直球和食莫逆(ばくげき)）
- ✅ ネットワークエラー時の自動フォールバック

## 技術的な実装詳細

### 使用した技術・API
| 技術 | 用途 |
|------|------|
| React 19 | フロントエンド フレームワーク |
| Tailwind CSS 4 | スタイリング |
| shadcn/ui | UI コンポーネント |
| tRPC 11 | 型安全な API |
| Express 4 | バックエンド フレームワーク |
| MySQL/TiDB | データベース |
| Drizzle ORM | ORM |
| Google Maps API | 地図、ジオコーディング |
| Nominatim API | 逆ジオコーディング |
| Open-Meteo API | 天気予報 |
| LLM API | AI メモ読み取り、場所候補検索 |
| S3 | ファイルストレージ |
| Manus OAuth | 認証 |

### 主要な問題と解決方法
| 問題 | 原因 | 解決方法 |
|------|------|---------|
| Google Maps URL リダイレクト取得がタイムアウト | Node.js fetch がブロック | curl フォールバック実装 |
| 割り勘の支払者が未選択でも追加可能 | バリデーション不足 | 支払者必須チェック追加 |
| 支出追加フォームがスマートフォンで画面外 | フォームが縦に長い | スクロール対応レイアウト |
| 天気予報が表示されない | Open-Meteo API fetch がブロック | curl フォールバック実装 |
| 座標特定の精度が低い | 全角数字の処理不足 | 全角→半角変換強化 |
| AI 場所候補検索で略称が出ない | プロンプトが不十分 | AI プロンプト改善 |
